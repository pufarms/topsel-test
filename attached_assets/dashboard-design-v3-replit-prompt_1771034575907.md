# 관리자 대시보드 디자인 개선 — 레이아웃 및 시각 구조 변경

## ⚠️ 최우선 원칙

1. **기존 CSS, 글씨체, 폰트를 그대로 사용하세요.** 새로운 폰트나 CSS 파일을 추가하지 마세요.
2. **각 섹션(컬럼)에 설정된 기존 기능, 버튼 동작, API 호출, 상태 관리, 이벤트 핸들러는 절대 수정하지 마세요.** 모든 기능과 역할이 100% 보존되어야 합니다.
3. **변경하는 것은 오직: 레이아웃 배치 순서, 배경색, 외곽선 두께/색상, 그림자, 패딩, 간격, 위치 구조뿐입니다.**
4. **함께 첨부된 예시 파일(dashboard-preview-v3-2.jsx)은 디자인 방향을 보여주는 참고용입니다. 똑같이 만들 필요 없습니다.** 기존 프로젝트의 컴포넌트 구조와 코드 스타일을 유지하면서, 이 프롬프트에 명시된 레이아웃/배경/간격 등의 시각적 변경사항만 반영하세요.
5. 작업 진행 상황과 결과는 항상 한글로 보여주세요.

## 변경 2: 전체 레이아웃 구조 변경

현재 대시보드의 섹션 배치 순서를 아래와 같이 변경합니다.
기존에 세로로 나열된 구조를 **4행 구조**로 재배치합니다.

```
현재 구조 (변경 전):
──────────────────
섹션들이 세로 1열로 나열

변경 후 구조:
──────────────────
1행: [주문/배송 현황] ──────────────────────── 전체 너비 1열
     (간격 32px)
2행: [회원 현황] | [문의 현황] ──────────────── 가로 2열 (50:50)
     (간격 32px)
3행: [매출 현황] ────────────────────────────── 전체 너비 1열
     (간격 32px)
4행: [행사진행 현황] | [공지사항(신규)] ──────── 가로 2열 (50:50)
```

- 각 행 사이의 간격: **32px** (`space-y-8`)
- 2열 배치 시 컬럼 사이 간격: **20px** (`gap-5`)
- 메인 콘텐츠 영역 패딩: **20px** (`p-5`)

## 변경 3: 각 섹션별 배경/외곽/그림자 스타일

### 1행 — 주문/배송 현황

- **배경색**: 연한 크림색 (`bg-amber-50`)
- **외곽선**: 두께 2px, 색상 amber-300 (`border-2 border-amber-300`)
- **그림자**: `shadow-lg`
- **둥근 모서리**: `rounded-xl`
- **패딩**: 좌우 20px, 상단 20px, **하단 48px** (`px-5 pt-5 pb-12`) — 하단 여백은 향후 버튼 배치 공간
- **섹션 타이틀 아래 간격**: 16px (`mb-4`)

**기간 필터 버튼 위치:**
- 우측 상단에 세로 2줄로 배치 (`flex flex-col items-end gap-1`)
- 1줄: "기간:" 라벨 + 버튼 7개 (오늘, 어제, 이번 주, 지난주, 이번 달, 지난달, 직접 선택)
- 2줄: 날짜 표시 "📅 2026-02-14 (오늘)" — 우측 정렬
- 선택된 버튼: `bg-blue-600 text-white shadow-sm`
- 미선택 버튼: `bg-white text-gray-600 border border-gray-300`

**내부 카드 7개 (가로 1줄):**
- 레이아웃: `flex gap-2`, 각 카드 `flex-1`
- 카드별 배경색과 라벨 (기존 사이트와 동일하게 맞춤):

| 순서 | 라벨 | 카드 배경 | 외곽선 | 숫자 색상 |
|------|------|----------|--------|----------|
| 1 | 전체주문 | bg-yellow-50 | border-yellow-200 | text-blue-700 |
| 2 | 주문대기 | bg-yellow-50 | border-yellow-200 | text-red-600 |
| 3 | 주문조정 | bg-yellow-50 | border-yellow-200 | text-blue-700 |
| 4 | 상품준비중 | bg-purple-50 | border-purple-200 | text-purple-700 |
| 5 | 배송준비중 | bg-blue-50 | border-blue-200 | text-blue-700 |
| 6 | 취합/취소 | bg-red-50 | border-red-200 | text-red-600 |
| 7 | 배송중 | bg-green-50 | border-green-200 | text-green-700 |

- 카드 내부 패딩: `p-3.5`
- 카드 내부 구조: 아이콘+라벨(상단) → 숫자값+건(하단)
- 배송준비중 카드에만 부가정보 텍스트 표시: "운송장미등록건 · 직접배송건전용"

⚠️ **카드의 라벨, 숫자 데이터, 클릭 이벤트 등 기존 기능은 그대로 유지하세요. 배경색과 배치만 변경합니다.**

### 3행 — 매출 현황

- **배경색**: 흰색 (`bg-white`)
- **하단 보더**: 두께 4px, 초록색 (`border-b-4 border-emerald-400`)
- **그림자**: `shadow-lg`
- **둥근 모서리**: `rounded-xl`
- **패딩**: `px-5 py-5`
- **섹션 타이틀**: 초록색 원형 배지 번호 "4"

**내부 구조 — 확정매출/예상매출을 좌우 2열 배치:**
```
[매출 현황]
┌──────────────────────┬──────────────────────┐
│ 확정매출              │ 예상매출              │
│ [금일][전일][전월][이번달] │ [금일][전일][전월][이번달] │
└──────────────────────┴──────────────────────┘
```
- 레이아웃: `grid grid-cols-2 gap-4`
- 확정매출 구분 라벨: `bg-emerald-500 text-white` 배지 + 설명 텍스트
- 예상매출 구분 라벨: `bg-gray-400 text-white` 배지 + 설명 텍스트
- 각 영역 내 카드 4개: `flex gap-2`
- 확정매출 금일 카드: `bg-emerald-500 text-white` (강조)
- 확정매출 나머지: `bg-emerald-50 border border-emerald-200`
- 예상매출 카드: `bg-gray-50 border border-gray-200`
- 카드 내부 패딩: `p-3.5`

⚠️ **매출 데이터 연동, 금액 계산 로직, 기간 필터 등 기존 기능은 그대로 유지하세요.**

⚠️ **행사 데이터 연동, 테이블 내용, 상태 표시 등 기존 기능은 그대로 유지하세요.**

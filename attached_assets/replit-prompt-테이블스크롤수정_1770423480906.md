# 작업 지시서: 모든 테이블 좌우 스크롤 항상 표시 (스크롤바 위치 수정)

## 📋 문제점
관리자와 회원 페이지의 주문 관련 테이블에서 좌우(수평) 스크롤 사용이 불편합니다.

**현재 문제:**
- 일부 테이블: 데이터 목록을 끝까지 아래로 스크롤해야만 좌우 스크롤바가 나타남
- 일부 테이블: 데이터를 다 내리지 않아도 스크롤바가 보임 → 일관성 없음
- 원인 추정: 수평 스크롤이 테이블 컨테이너(wrapper) 바깥에 있거나, `overflow-x` 설정이 테이블 전체 높이를 포함하는 외부 요소에 적용되어 있어서, 페이지를 맨 아래까지 스크롤해야 수평 스크롤바에 도달하는 구조

**원하는 동작:**
- 모든 테이블에서 좌우 스크롤바가 **항상 보이고 사용 가능**해야 함
- 데이터가 많아도 아래로 끝까지 스크롤하지 않고도 좌우 스크롤 가능

## 🔧 구현 방법

### 핵심: 테이블 컨테이너에 고정 높이 + overflow 설정

**올바른 구조:**
```
┌─────────────────────────────────────────┐
│ 필터 영역 (날짜, 카테고리, 검색 등)         │
├─────────────────────────────────────────┤
│ ┌─────────────────────────────────────┐ │
│ │ 테이블 헤더 (고정)                    │ │  ← 세로 스크롤해도 헤더는 고정
│ ├─────────────────────────────────────┤ │
│ │                                     │ │
│ │ 테이블 데이터 (세로 스크롤 영역)       │ │  ← 이 영역만 세로 스크롤
│ │                                     │ │
│ │                                     │ │
│ ├─────────────────────────────────────┤ │
│ │ ◄ ■■■■■■□□□□□□□□□□□□□□□□□□□□ ►    │ │  ← 좌우 스크롤바: 테이블 바로 아래
│ └─────────────────────────────────────┘ │
│ 페이지네이션 / 표시 개수 선택              │
└─────────────────────────────────────────┘
```

**잘못된 구조 (현재 일부 테이블):**
```
┌─────────────────────────────────────────┐
│ 필터 영역                                │
│ 테이블 헤더                              │
│ 데이터 행 1                              │
│ 데이터 행 2                              │
│ ...                                     │
│ 데이터 행 50                             │  ← 여기까지 페이지 스크롤해야
│ 데이터 행 100                            │
│ ◄ ■■■■■■□□□□□□□□□□□□□□ ►              │  ← 좌우 스크롤바가 여기에 있음
│ 페이지네이션                              │
└─────────────────────────────────────────┘
```

### CSS/스타일 수정 방법

**방법: 테이블 래퍼에 max-height + overflow 설정**

프로젝트에서 테이블을 감싸는 컨테이너(wrapper) 컴포넌트를 찾아서 다음과 같이 수정:

```css
/* 테이블을 감싸는 컨테이너 */
.table-container {
  overflow-x: auto;      /* 좌우 스크롤 항상 가능 */
  overflow-y: auto;      /* 세로 스크롤도 컨테이너 내에서 */
  max-height: 70vh;      /* 화면 높이의 70% (조정 가능) */
  position: relative;
}

/* 테이블 헤더 고정 (sticky) */
.table-container thead th {
  position: sticky;
  top: 0;
  z-index: 10;
  background: white;     /* 다크모드 대응 필요 */
}

/* 스크롤바 항상 표시 (선택사항) */
.table-container::-webkit-scrollbar {
  height: 8px;           /* 수평 스크롤바 높이 */
  width: 8px;            /* 수직 스크롤바 너비 */
}
.table-container::-webkit-scrollbar-thumb {
  background: #c1c1c1;
  border-radius: 4px;
}
.table-container::-webkit-scrollbar-track {
  background: #f1f1f1;
}
```

### 공통 컴포넌트로 통일

**프로젝트에서 테이블 관련 공통 컴포넌트가 있을 가능성이 높습니다.**

shadcn/ui의 `Table` 컴포넌트를 사용하고 있다면, 테이블을 감싸는 `ScrollArea` 또는 래퍼 `div`를 찾아서 수정하세요.

**전체 프로젝트 검색으로 테이블 컨테이너 찾기:**
```bash
# 프로젝트에서 테이블 관련 overflow 설정 검색
grep -r "overflow-x" client/src/
grep -r "overflow" client/src/components/ui/table
grep -r "ScrollArea" client/src/
grep -r "table-container" client/src/
grep -r "overflow-auto" client/src/
```

**가능한 수정 위치들:**
1. `client/src/components/ui/table.tsx` - shadcn/ui 테이블 컴포넌트
2. 각 페이지에서 테이블을 감싸는 `<div className="overflow-auto">` 또는 유사한 래퍼
3. 공통 테이블 래퍼 컴포넌트가 있다면 그것을 수정

**Tailwind CSS 클래스로 수정하는 경우:**
```jsx
{/* 수정 전 - 문제가 있는 구조 */}
<div className="overflow-x-auto">
  <Table>
    {/* 수백 줄의 데이터... 끝까지 내려야 수평 스크롤바 보임 */}
  </Table>
</div>

{/* 수정 후 - 올바른 구조 */}
<div className="overflow-auto max-h-[70vh] relative">
  <Table>
    <TableHeader className="sticky top-0 z-10 bg-white dark:bg-gray-950">
      {/* 헤더 고정 */}
    </TableHeader>
    <TableBody>
      {/* 데이터 행들 */}
    </TableBody>
  </Table>
</div>
```

## ⚠️ 주의사항

1. **모든 테이블에 일관되게 적용**: 관리자/회원 모든 페이지의 테이블을 검색하여 하나도 빠짐없이 적용
2. **테이블 헤더(thead) 고정 유지**: 현재 이미 헤더 고정이 적용된 테이블이 있다면 그 기능 유지
3. **다크 모드 대응**: 헤더 고정 시 배경색이 다크 모드에서도 올바르게 표시
4. **반응형 대응**: 모바일/태블릿에서도 좌우 스크롤이 정상 작동
5. **max-height 값 조정**: 70vh는 기본값이며, 각 페이지의 필터 영역 높이에 따라 적절히 조정 가능. 테이블 위에 필터가 많은 페이지는 60vh, 적은 페이지는 75vh 등으로 조정 가능
6. **표시 개수 선택(10/30/100/전체)**: 전체 선택 시 데이터가 매우 많아질 수 있으므로 max-height와 스크롤이 더욱 중요
7. **기존 기능 유지**: 테이블 정렬, 체크박스 선택, 행 클릭 이벤트 등 기존 기능에 영향 없어야 함

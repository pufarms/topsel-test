# 📦 Topsel B2B 디자인 시스템 - 사용 가이드

## 🎉 완성된 패키지 구성

이 디자인 시스템은 **Replit이 모든 페이지 작업 시 반드시 따라야 할 완전한 가이드**입니다.

---

## 📂 포함된 파일

### 1. **design-system-responsive.json** (핵심 규칙)
- 색상 팔레트 (Brand, Neutral, Semantic)
- 6단계 반응형 Breakpoints (xs ~ 2xl)
- 타이포그래피 위계 (반응형 크기 포함)
- UI 컴포넌트 정의 (버튼, 카드, 아이콘)
- 섹션 패턴 (Hero, Feature Cards, Content+Image 등)
- 성능 최적화 가이드
- **Replit 필수 준수 지시사항**

### 2. **design-system-global.css** (글로벌 스타일시트)
- 모든 CSS Variables 정의 (:root)
- 기본 Reset & Base Styles
- 반응형 타이포그래피 클래스
- 섹션 배경 클래스 (.section-light, .section-dark)
- 버튼 스타일 (.btn-primary, .btn-outline-white)
- 카드 컴포넌트 (.card)
- 그리드 시스템 (.grid)
- 접근성 & 유틸리티 클래스

**⚠️ 중요**: 이 CSS는 **모든 페이지의 `<head>` 섹션에 반드시 포함**해야 합니다!

### 3. **REPLIT-STRICT-GUIDELINES.md** (엄격한 준수 사항)
- 작업 시작 전 체크리스트
- 색상 사용 규칙 (허용/금지)
- 타이포그래피 규칙
- 섹션 배경 교차 패턴 (White ↔ Navy)
- 반응형 Breakpoint 테스트
- 버튼/카드 스타일 가이드
- 레이아웃 패턴 예시
- **작업 완료 전 최종 체크리스트**
- 절대 금지 사항
- 모범 사례

### 4. **page-template.html** (실전 템플릿)
- 완전한 HTML 페이지 예시
- Header (Sticky, 반응형 네비게이션)
- Hero Section (Navy 배경, 통계 포함)
- Feature Cards Grid (1→2→3 단 반응형)
- Content + Image Section (이미지/텍스트 교차)
- CTA Banner (Gradient 배경)
- Footer (다단 레이아웃)
- 디자인 시스템 100% 준수

## 🚀 Replit 사용 방법

### **Step 1: 파일 업로드**
```
1. Replit 프로젝트에 모든 파일 업로드
2. design-system-global.css를 프로젝트 루트에 배치
3. design-system-responsive.json을 참고 문서로 보관
```

### **Step 2: 모든 HTML 페이지에 CSS 링크**
```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>페이지 제목</title>
  
  <!-- ⚠️ 필수: 글로벌 디자인 시스템 -->
  <link rel="stylesheet" href="design-system-global.css">
</head>
<body>
  <!-- 페이지 내용 -->
</body>
</html>
```

### **Step 3: 새 페이지 작성 시 절차**
```
1. REPLIT-STRICT-GUIDELINES.md 읽기
2. design-system-responsive.json에서 필요한 section_pattern 선택
3. page-template.html을 기반으로 시작
4. 정의된 클래스만 사용 (.h1-hero, .btn-primary, .card, 등)
5. 섹션 배경을 White ↔ Navy로 교차
6. 375px, 768px, 1024px에서 테스트
7. 최종 체크리스트 확인
```

## ⚠️ 절대 규칙 (Replit이 절대 어기면 안 되는 것)

### 1. **색상**
❌ 금지: `color: #123456;` (임의 색상)
✅ 허용: `color: var(--primary);` (정의된 변수)

### 2. **폰트 크기**
❌ 금지: `<h1 style="font-size: 60px;">` (임의 크기)
✅ 허용: `<h1 class="h1-hero">` (정의된 클래스)

### 3. **섹션 배경**
❌ 금지: Navy → Navy 연속
✅ 허용: Navy → White → Navy

### 4. **버튼**
❌ 금지: 새로운 버튼 스타일 생성
✅ 허용: `.btn-primary`, `.btn-outline-white`, `.btn-pill`만 사용

### 5. **반응형**
❌ 금지: `@media (max-width: 768px)` (Desktop-first)
✅ 허용: `@media (min-width: 768px)` (Mobile-first)

## 📞 문제 해결

### Q: 디자인 시스템에 없는 요소가 필요하면?
A: 절대 임의로 만들지 말고, 사용자에게 물어보세요:
```
"이 [요소]는 디자인 시스템에 정의되어 있지 않습니다.
기존 스타일 중 어떤 것을 사용할까요?
아니면 새로 추가가 필요할까요?"
```

### Q: 반응형이 제대로 작동하지 않으면?
A: REPLIT-STRICT-GUIDELINES.md의 "반응형 브레이크포인트" 섹션 확인

### Q: 섹션 배경 순서가 헷갈리면?
A: page-template.html을 참고하세요 (Header → Hero(Navy) → Features(White) → Content(Navy) → ...)

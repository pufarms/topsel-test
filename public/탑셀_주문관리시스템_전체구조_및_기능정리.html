탑셀 주문관리시스템 - 전체 구조 및 기능 정리

작성일: 2026년 2월 6일

1. 시스템 개요

한국어 기반의 주문관리시스템으로, 판매회원(셀러)과 관리자(어드민)를 위한 별도 대시보드를 제공합니다. 회원의 주문 접수부터 배송 완료까지의 전 과정을 체계적으로 관리하며, 상품/재고/정산/통계 등 운영에 필요한 종합 기능을 갖추고 있습니다.

기술 스택

구분기술
프론트엔드React 18, TypeScript, Vite, Tailwind CSS, shadcn/ui, Wouter(라우팅), TanStack Query
백엔드Express 5, Node.js, TypeScript
데이터베이스PostgreSQL + Drizzle ORM
인증세션 기반 (express-session) + SHA256 해싱
실시간 통신SSE (Server-Sent Events)
이미지 저장Cloudflare R2 (S3 호환)
이메일Resend API
주소 검증행정안전부 주소 API + Anthropic Claude AI

1. 대기 → 2. 상품준비중 → 3. 배송준비중 → 4. 배송중
단계상태관리 페이지역할

2단계상품준비중 (운송장 출력)preparing.tsx운송장 입력/출력 후 배송준비중으로 전환 (자동/수동)

4단계배송중shipping.tsx배송 진행 중

상태관리 페이지설명

3-3. 배송준비중 단계 상세 (핵심 워크플로우)

중요! 배송준비중 단계에서의 가시성 제어 및 버튼 워크플로우

가시성 제어

  • 상품준비중 → 배송준비중으로 전환된 주문은 관리자 화면에서만 보임
  • 관리자가 "운송장 전달" 버튼을 클릭하면 회원 화면에서도 보이기 시작
  • waybill_delivered 설정으로 제어 (site_settings 테이블)

버튼 워크플로우

버튼작동 조건완료 후 상태
운송장 전달독립 작동 (항상 활성)버튼 텍스트 → "운송장전달 완료", 회원에게 주문 노출
회원취소마감독립 작동 (운송장 전달과 별개)회원 취소 접수 마감, 배송중 전송 가능
배송중 전송취소마감 완료 필요배송준비중 → 배송중으로 전환, 설정 초기화

회원 취소 처리

  • 회원이 취소 요청 시 즉시 처리 (관리자 승인 불필요)
  • 취소 시 재고 자동 복원
  • 취소마감 후에는 회원 취소 불가

8. 핵심 비즈니스 규칙

8-1. 상품가격 체계

  • 현재공급가(current_products)가 모든 기준 - 주문, 정산, 통계의 기준
  • 상품등록 → 1원 단위 정밀 계산
  • 전송 시 10원 올림 적용: Math.ceil(price/10)*10
  • 회원 등급별 공급가: start / driving / top 등급에 따라 다른 가격 적용

8-2. 주문 규칙

  • 주문 시 반드시 현재공급가 테이블에서 상품 존재 여부 확인
  • 공급중지 상품은 주문 불가
  • 동일 주문은 4단계 중 하나에만 존재 (중복 불가)
  • 모든 주문은 대기 → 상품준비중 → 배송준비중 → 배송중 순서로 진행

8-3. 주소 검증 시스템

  • 엑셀 주문 업로드 시 다단계 주소 검증 파이프라인 적용
  • 정규식, 패턴 유사도, 규칙 기반 검증 (Juso.go.kr API)
  • AI 정규화 (Anthropic Claude AI) + 학습 메커니즘
  • 주소 상태: VALID WARNING INVALID

9. 실시간 통신 (SSE)

서버에서 클라이언트로 실시간 이벤트를 전송하여 주문 상태 변경을 즉시 반영합니다.

  • SSE는 알림 전용 - 실제 데이터는 보안된 API를 통해 가져옴
  • 관리자/회원 각각 별도 SSE 채널
  • 주문 생성, 수정, 삭제, 상태 변경 시 이벤트 발송
  • 클라이언트에서 이벤트 수신 시 해당 쿼리 자동 갱신 (TanStack Query 무효화)

10. 디자인 및 UI 원칙

  • 반응형 웹 디자인: 모바일/태블릿/데스크톱 대응
